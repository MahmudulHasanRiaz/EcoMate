P47bp Mini Fix - Purchase check-status result guard (must not silently pass)

Scope
- `src/app/api/check-passing/route.ts` only
- No unrelated refactor
- No commit/push instructions

Problem
- Purchase branch now calls `updateCheckStatusCore(...)`, but does not verify returned `{ success }`.
- If core returns `{ success: false, message }` (without throw), API may still return success.

Required change
1) In Purchase section (inside PATCH loop), replace direct await call with guarded result:

```ts
const { updateCheckStatusCore } = await import('@/server/modules/purchases');
const purchaseResult = await updateCheckStatusCore({
  paymentId: update.id,
  status,
  user: actor.name,
});

if (!purchaseResult?.success) {
  throw new Error(
    purchaseResult?.message
      ? `Purchase reconcile failed (${update.id}): ${purchaseResult.message}`
      : `Purchase reconcile failed (${update.id})`
  );
}
```

2) Keep existing behavior unchanged for:
- Expense (`updateExpenseCheckStatus`)
- Staff (`updateStaffPaymentCheckStatus`)
- logs creation
- response shape

Acceptance
- If purchase core reconciliation fails, PATCH must fail with clear error (no false success).

Verification
1) Static grep
```bash
rg -n "purchaseResult|reconcile failed|updateCheckStatusCore" src/app/api/check-passing/route.ts
```

2) Manual failure simulation (if possible in local):
