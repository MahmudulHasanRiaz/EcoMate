P47ar - Order status update fails (assignedToId + logs)

Problem
- Cancel/Update shows "Could not update status".
- Prisma error: Unknown argument `assignedToId` + invalid `logs` relation in order.update.

Scope
- src/server/modules/orders.ts

Fix (apply exactly)

1) Replace `logs` with `OrderLog` in order.update
In updateOrderDetails (around the transaction update):
```
logs: { create: logEntry }
```
=> change to:
```
OrderLog: { create: logEntry }
```

Also search the same file for any other nested writes using `logs:` and replace with `OrderLog:` to match schema.

2) Use relation `assignedTo` instead of scalar `assignedToId` in updates
In updateOrderDetails, before tx.order.update, normalize assignment:

```
if (has('assignedToId')) {
  if (payload.assignedToId) {
    updateData.assignedTo = { connect: { id: payload.assignedToId } };
  } else {
    updateData.assignedTo = { disconnect: true };
  }
  delete (updateData as any).assignedToId;
}

if (shouldAutoAssign && actor.id) {
  updateData.assignedTo = { connect: { id: actor.id } };
}
```

This avoids Prisma errors even if assignedToId is missing in the client.

3) Prisma client + restart
```
npx prisma generate
npm run dev
npm run worker
```

Verify
- Cancel/Status update should succeed.
- No "Unknown argument assignedToId" or "logs" errors.
