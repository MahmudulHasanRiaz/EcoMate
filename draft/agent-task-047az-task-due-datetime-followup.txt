P47az-followup - Ensure migration is applied (no db push)

Why
- dueDate type change already in schema + UI.
- migration file exists but must be applied/resolved locally so history is correct.

Local steps (coder)
1) Stop dev server (avoid file lock).
2) Try migration:
   npx prisma migrate dev --name task_due_datetime

If it fails with P1014 (shadow DB drift):
3) Keep the existing migration file and mark it applied:
   npx prisma migrate resolve --applied 20260131165530_task_due_datetime
4) Regenerate client:
   npx prisma generate
5) Restart dev server.

Notes
- Do NOT use prisma db push for this.
- This keeps migration history consistent for production deploy.
