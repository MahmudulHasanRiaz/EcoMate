==================================================
A) Findings from Audit (must fix)
==================================================

1) Large batch warning is not rendered (claim mismatch)
File:
- `src/app/dashboard/orders/client-page.tsx`

Issue:
- State exists:
  - `printWarning`
  - `isBulkPrinting`
- Trigger exists in `handleBulkPrint` for `selectedOrders.length > 300`.
- But no AlertDialog/UI is rendered for `printWarning`.
- Result: warning never shown.

Required fix:
- Add an `AlertDialog` tied to `printWarning.isOpen`:
  - Title: "Large print batch"
  - Message: browser may be slow for very large batches.
  - Actions:
    - Cancel: close dialog
    - Continue: call `executeBulkPrint(printWarning.type!)`

Acceptance:
- Selecting >300 and clicking bulk print must show warning dialog before printing.

--------------------------------------------------

2) "Up to 2000 supported" is not operationally safe with current fetch pattern
File:
- `src/app/print/bulk/client-page.tsx`

Issue:
- Current loading still does N+1:
  - `Promise.all(orderIds.map(id => getOrderById(id)))`
- For 500/1000/2000 this can overwhelm browser/network and fail partially.

Required fix:
- Add new API for bulk order print payload:
  - `POST /api/print/bulk/orders`
  - Input: `{ ids: string[] }`
  - Permission: enforce `orders:read`
  - Output: minimal order fields needed by `InvoiceTemplate`/`StickerTemplate`.
- Update print page to use this single bulk endpoint instead of per-order requests.
- Keep old per-order fallback only if bulk endpoint fails.

Acceptance:
- Print page performs one primary request for batch data, not thousands.
- 120+ loads significantly faster and more reliably.

--------------------------------------------------

3) Print token routes should enforce order read permission, not just auth
Files:
- `src/app/api/print/bulk/route.ts`
- `src/app/api/print/bulk/[token]/route.ts`

Issue:
- Routes currently check only `currentUser`.
- Must align with order data sensitivity model.

Required fix:
- Replace/augment auth check with `enforcePermission('orders','read')`.
- Keep existing token TTL and response shapes.

Acceptance:
- User without order-read permission cannot create/retrieve print batches.
