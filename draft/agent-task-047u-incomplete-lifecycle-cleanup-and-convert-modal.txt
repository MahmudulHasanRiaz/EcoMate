P47u - Incomplete Leads Lifecycle Cleanup + Convert UX + Not Converted Status

Why this is needed (tell coder)
- Leads can remain OPEN even after a real order exists (e.g., webhook/complete call missed).
- Convert is one-click; we need an edit modal.
- We need NOT_CONVERTED (analytics) instead of only delete/cancel.

Scope
- Minimal schema impact: status is String, so no migration needed.
- Files: ONLY the ones listed below.

------------------------------------------------------------
1) Auto-close OPEN leads when a real order already exists (server-side cleanup)
File: src/app/api/orders/incomplete/route.ts

Add a cleanup step BEFORE building payload:

Implementation outline:

```ts
const TTL_MINUTES = 60; // keep modest; can tune later
const cutoff = new Date(Date.now() - TTL_MINUTES * 60 * 1000);

// 1) load open leads (already present)
const leads = await prisma.wooCheckoutLead.findMany({ ... status: 'OPEN' ... });

// 2) build phone+business sets
const phones = Array.from(new Set(leads.map(l => l.phoneNormalized).filter(Boolean)));
const businesses = Array.from(new Set(leads.map(l => l.businessId).filter(Boolean)));

// 3) find recent real orders matching these phones+businesses
let recentOrders: Array<{ businessId: string; customerPhone: string }> = [];
if (phones.length && businesses.length) {
  recentOrders = await prisma.order.findMany({
    where: {
      businessId: { in: businesses },
      customerPhone: { in: phones },
      createdAt: { gt: cutoff },
      status: { notIn: ['Cancelled', 'Canceled'] },
    },
    select: { businessId: true, customerPhone: true }
  });
}

// 4) mark matching leads as COMPLETED
if (recentOrders.length) {
  const keySet = new Set(recentOrders.map(o => `${o.businessId}::${o.customerPhone}`));
  const toClose = leads.filter(l => l.phoneNormalized && keySet.has(`${l.businessId}::${l.phoneNormalized}`));
  if (toClose.length) {
    await prisma.wooCheckoutLead.updateMany({
      where: { id: { in: toClose.map(l => l.id) } },
      data: { status: 'COMPLETED', completedAt: new Date() }
    });
  }
}

// 5) re-query OPEN leads after cleanup (or filter in-memory)
const finalLeads = toClose?.length
  ? await prisma.wooCheckoutLead.findMany({ ... same query ... })
  : leads;
```

Then use `finalLeads` to build payload.

2) Add NOT_CONVERTED action (analytics-friendly terminal state)
File: src/app/api/orders/incomplete/[id]/route.ts

Update PATCH handler:
- allow actions: cancel, not_converted
- map:
  - cancel -> status: 'CANCELLED'
  - not_converted -> status: 'NOT_CONVERTED'
- both set completedAt: new Date()

Suggested patch:

```ts
const action = String(body?.action || 'cancel').toLowerCase();
const statusMap: Record<string, string> = {
  cancel: 'CANCELLED',
  not_converted: 'NOT_CONVERTED',
};
const nextStatus = statusMap[action];
if (!nextStatus) return apiError('Unsupported action', 400);

const lead = await prisma.wooCheckoutLead.update({
  where: { id: params.id },
  data: { status: nextStatus, completedAt: new Date() },
});
```

Helper:
```ts
function skuListToCsv(list: any[] | undefined) {
  if (!Array.isArray(list)) return '';
  return list.map((s: any) => (typeof s === 'string' ? s : (s?.sku || s?.SKU || ''))).filter(Boolean).join(', ');
}
```
