UPDATE P40j - Add shared webhook-management lock (prevent ensure vs autoheal race)

Scope: src/server/modules/woo-sync.ts + src/app/api/webhooks/woo/[id]/route.ts only.

Problem:
- Auto-heal can run while manual sync is about to call ensureWooWebhook.
- Even with recent/inflight guards, a race still allows double create when auto-heal fires first.

Fix:
1) Create a shared lock key for webhook management
Key: woo:webhook:manage:<integrationId>
TTL: 90s (short safety window)

2) Acquire lock in ensureWooWebhook
File: src/server/modules/woo-sync.ts
- Add helper:
```
async function tryAcquireWooWebhookLock(integrationId: string) {
  const redis = getRedisClient();
  if (redis) {
    const ok = await redis.set(`woo:webhook:manage:${integrationId}`, '1', 'NX', 'EX', 90);
    return ok === 'OK';
  }
  // memory fallback
  const now = Date.now();
  const last = wooWebhookManageMap.get(integrationId);
  if (last && now - last < 90 * 1000) return false;
  wooWebhookManageMap.set(integrationId, now);
  return true;
}

async function releaseWooWebhookLock(integrationId: string) {
  const redis = getRedisClient();
  if (redis) await redis.del(`woo:webhook:manage:${integrationId}`);
  wooWebhookManageMap.delete(integrationId);
}
```
- Add module map near top:
  const wooWebhookManageMap = new Map<string, number>();

- In ensureWooWebhook (start):
```
if (!(await tryAcquireWooWebhookLock(integration.id))) {
  console.log('[WOO_WEBHOOK_MANAGE_LOCK_BUSY]', integration.id);
  return;
}
```
- In finally block:
```
await releaseWooWebhookLock(integration.id);
```

3) Use same lock in auto-heal
File: src/app/api/webhooks/woo/[id]/route.ts
- Add helper (similar to above) OR import from a small shared util if you prefer.
- Before any autoheal work in triggerWooWebhookAutoHeal:
```
if (!(await tryAcquireWooWebhookLock(integrationId))) {
  console.log('[WOO_WEBHOOK_MANAGE_LOCK_BUSY]', integrationId);
  return;
}
```
- After ensureWooWebhook call (or in finally), release the lock.
