P47bs - Staff Fine feature with ledger-safe accounting integration

Objective
- Add ability to record staff fines from staff profile.
- Show fine list/history in staff profile.
- Keep staff due and accounting fully consistent.

Important rules
- Fine is a deduction from staff payable (not a cash transaction).
- Only ACTIVE fines affect due and ledger.
- Voided fine must be excluded from due and ledger.

2) Server module (new): staff-fines.ts
--------------------------------------------------------------------

Create `src/server/modules/staff-fines.ts`.

Required functions
1. `listStaffFinesPaginated(params)`
- inputs: `{ staffId, cursor?, pageSize? }`
- returns `{ items, nextCursor }`
- order: date desc, id desc

2. `getActiveFineTotalForStaff(staffId, tx?)`
- returns sum(amount) where status = Active

3. `batchGetActiveFineTotals(staffIds, tx?)`
- returns `Map<staffId, totalFine>` for list API performance

4. `rebuildStaffFineLedger(tx, fineId)`
- postingGroup = `staffFine:<fineId>`
- delete existing entries for this postingGroup first
- if fine is not Active => return (no ledger rows)
- account mapping:
  - debit account: `Salaries Payable` (`ACCOUNT_LABELS.salariesPayable`)
  - credit account: `Salary Expense` (`ACCOUNT_LABELS.salary`)
- create 2 ledger rows:
  - Dr Salaries Payable = fine.amount
  - Cr Salary Expense = fine.amount
- use `resolveLedgerEntryNumber` for entryNumber
- description: `Staff fine: <staff name> - <reason>`

5. `createStaffFineCore(payload, tx)`
- validate amount > 0
- validate staff exists
- current due = max(0, totalEarned - totalPaid - totalActiveFine)
  - totalEarned from `StaffIncome` sum
  - totalPaid from cleared-funds logic (reuse existing helper in staff module)
- block if `amount > currentDue` with explicit message
- create fine row with status Active
- call `rebuildStaffFineLedger`

6. `voidStaffFineCore(fineId, actor, tx)`
- set status = Voided, set voided metadata
- call `rebuildStaffFineLedger` (which will remove entries)
