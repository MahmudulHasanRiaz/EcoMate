P47dh - Woo Webhook No-Hit (Local Tunnel) Fix Plan

Context (do not ignore)
- Environment is LOCAL machine with Cloudflare tunnel domain: `https://dev.panjabiclub.com`.
- This is NOT `/opt/fashionary` production command flow.
- Current evidence:
  - `npx tsx scripts/list-woo-webhooks.ts` shows one active `order.created` webhook pointing to `https://dev.panjabiclub.com/api/webhooks/woo/<integrationId>`.
  - App logs do NOT show `POST /api/webhooks/woo/<integrationId>` during real order creation.
  - Logs show only `/woo/api/incomplete-orders*` and `/woo/api/restriction-check`, which are different endpoints.

Objective
- Restore reliable webhook hit + auto-ingest in local tunnel flow.
- Keep strict signature security.
- Prevent webhook churn/recreate storms from unsigned probes.

Scope (strict)
- `src/app/api/webhooks/woo/[id]/route.ts`
- `src/server/modules/woo-sync.ts`
- `scripts/list-woo-webhooks.ts` (only if needed for better evidence)
- New script: `scripts/send-woo-signed-probe.ts`

Do not touch unrelated modules.

--------------------------------------------------
1) Route hardening for “no-hit” debugging + safe auto-heal
--------------------------------------------------
File: `src/app/api/webhooks/woo/[id]/route.ts`

Required changes:
1. Add entry log at top of POST handler:
   - `[WOO_WEBHOOK_HIT] ID=<integrationId> hasSignature=<bool> topic=<header or empty> ua=<short user-agent>`
   - This is mandatory for proof of incoming requests.

2. Add safe auto-heal gating helper logic:
   - Auto-heal is allowed only when request looks like real Woo webhook.
   - Use at least one of:
     - `x-wc-webhook-topic`
     - `x-wc-webhook-event`
     - `x-wc-webhook-source`
   - If none exist, treat as generic unsigned probe: return 401 without recreate.

3. Add cooldown for auto-heal attempts:
   - Per integration + reason (`missing-signature`, `sig-mismatch`, `missing-secret`).
   - 10 minutes TTL minimum.
   - Prefer Redis key if available, fallback to in-memory map.

4. Missing signature behavior:
   - Keep strict 401 + recordWebhookFailure.
   - Trigger auto-heal only if request passes Woo-header gate and cooldown.
   - Do NOT rotate secret for missing signature path.

5. Signature mismatch behavior:
   - Keep strict 401 + recordWebhookFailure.
   - Trigger auto-heal with `forceRecreate: true, rotateSecret: true` only when cooldown allows.

6. Missing secret behavior:
   - Keep strict 401 + recordWebhookFailure.
   - Trigger auto-heal with rotateSecret true when cooldown allows.

Acceptance for this section:
- Generic random probes cannot repeatedly recreate webhook.
- Real Woo-like failed requests can still self-heal.
- Logs clearly show webhook hit attempts.

--------------------------------------------------
2) Ensure flow must not thrash under repeated calls
--------------------------------------------------
File: `src/server/modules/woo-sync.ts`

Required changes:
1. Keep current base URL resolution and target logging.
2. Keep deletion/recreate logic for stale/inactive hooks.
3. Add/keep guard so repeated ensure requests do not recreate endlessly for the same integration in a short interval unless explicitly required.
4. Keep secret persistence only after successful create/ensure outcome.

Acceptance:
- No rapid recreate loop from repeated bad requests.
- `WOO_WEBHOOK_TARGET` remains visible and correct.

--------------------------------------------------
3) Add deterministic signed probe script (mandatory)
--------------------------------------------------
New file: `scripts/send-woo-signed-probe.ts`

Script requirements:
1. Load active integration (or optional arg integrationId).
2. Resolve base URL from env in this order:
   - `WOO_WEBHOOK_BASE_URL`
   - `NEXT_PUBLIC_APP_URL`
   - `APP_URL`
3. Build payload:
   - include random numeric/string `id` (e.g. `TEST_<timestamp>`).
4. Compute signature exactly like Woo:
   - HMAC SHA256 over raw JSON string with `integration.webhookSecret`, digest base64.
5. POST to `/api/webhooks/woo/<integrationId>` with headers:
   - `content-type: application/json`
   - `x-wc-webhook-signature: <computed>`
   - `x-wc-webhook-topic: order.created`
6. Print status code + body.

Acceptance:
- Script returns 200 on healthy path.
- App logs show `[WOO_WEBHOOK_HIT]` for this request.
