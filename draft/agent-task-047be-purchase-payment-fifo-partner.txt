P47be - Purchases: Remove inline payments, move to Partner FIFO payments + advance credit

Goal
- Purchases create/update should NOT take payments directly.
- Payments happen ONLY from Partner profile (or 3‑dot menu on partner list).
- Partner page shows: Previous Due + New Bills + Total Due (latest).
- FIFO allocation:
  - General purchases: by PurchaseOrder.date (oldest first)
  - Production steps: by ProductionStep.createdAt (oldest first)
- Overpayment becomes Advance Credit and is auto-applied to future bills (FIFO).
- Accounting stays correct (ledger entries for invoice, payment, and credit usage).

Scope
- Server: src/server/modules/purchases.ts, src/services/partners.ts
- UI: src/app/dashboard/purchases/new/page.tsx, src/app/dashboard/purchases/[id]/client-page.tsx

=== 3) Partner payment API (FIFO allocation) ===
New API route:
  POST /api/partners/payments
Payload:
  {
    partnerId: string,
    partnerType: 'supplier' | 'vendor',
    amount: number,
    method: 'Cash' | 'Bank' | 'bKash' | 'Nagad' | 'Rocket' | 'Check' | 'Direct',
    paidFromAccountId: string,
    checkNo?: string,
    checkDate?: string,
    note?: string
  }

Server logic (new function: applyPartnerPaymentCore):
- Validate amount > 0, accountId present, checkNo+checkDate required for checks.
- Determine FIFO targets:
  Supplier:
    - PurchaseOrder where supplierId=partnerId, type='general',
      status != 'Cancelled', paymentStatus in ('Unpaid','Partial')
    - order by date asc, id asc
  Vendor:
    - ProductionStep where vendorId=partnerId, costAmount > paidAmount,
      and parent PO status != 'Cancelled'
    - order by ProductionStep.createdAt asc (fallback to PO.date if null)

- Compute due per target:
  Supplier PO due = po.total - sum(payments cash + passed checks + credit-applied)
  Vendor step due = step.costAmount - step.paidAmount

- Apply existing creditBalance first:
  while creditBalance > 0 and target due > 0:
    allocate credit to oldest target
    create PurchasePayment row with:
      paymentMethod: 'Credit'
      cash: creditAmount (store as cash for tracking)
      check: 0
      poId + (productionStepId if vendor)
    Ledger entry:
      Dr Accounts Payable, Cr Supplier Advance (amount)
    decrement creditBalance

- Apply incoming payment amount FIFO:
  allocate across targets until amount=0:
    create PurchasePayment row with paymentMethod=method
    set cash/check fields
    set productionStepId for vendor, paymentFor = stepType
    for Supplier PO: paymentFor='General'
    ledger entry:
      Dr Accounts Payable, Cr paidFromAccount
      NOTE: if method=Check and status=Pending -> ledger amount = 0 (keep same current rule)

- If payment amount still remains after all dues:
    add to partner.creditBalance (advance)
    ledger entry:
      Dr Supplier Advance (Asset), Cr paidFromAccount

- After each allocation:
    syncStepPaidFromPaymentsTx(tx, poId)
    recomputePaymentStatusTx(tx, poId)

- Revalidate tags: ['purchases', 'partners', `partner:${partnerId}`]
