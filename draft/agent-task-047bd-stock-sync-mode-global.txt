P47bd - Global Stock Sync Mode (Inventory vs Published)

Goal (Global)
- Add ONE global setting “Stock Sync Mode: Yes/No” (not per-integration).
- YES (inventory-based) = current behavior (instock/outofstock from qty).
- NO (published-based) = Woo stock_status uses Product.isPublished (true => instock, false => outofstock).
- In NO mode: orders can be created/imported as New without stock.
- Stock reservation + stock deduction MUST still happen on Confirm (same as before).
- Combo rules: use combo’s own publish state for Woo status; Confirm must ensure ALL components have stock (if any component missing, block Confirm).

Scope
- UI: src/app/dashboard/settings/integrations/page.tsx (Woo settings page)
- Settings store: appSetting key "general" (JSON)
- Logic: src/server/modules/stock-sync.ts, src/server/modules/orders.ts, src/server/modules/woo-sync.ts
- Optional helper: src/server/utils/app-settings.ts
- Git: add /public/uploads/ to .gitignore

3) Apply mode to Woo stock push (ALL entry points)
File: src/server/modules/stock-sync.ts
- Add helper:
  async function getStockSyncMode(): Promise<'inventory' | 'publish'> {
    const settings = await getGeneralSettings();
    return settings.stockSyncMode === 'publish' ? 'publish' : 'inventory';
  }
- When mode === 'publish':
  - Status MUST be based on Product.isPublished (NOT qty).
  - For variable products, use parent product.isPublished for variations.
  - For combo products, use parent product.isPublished for its SKU.
