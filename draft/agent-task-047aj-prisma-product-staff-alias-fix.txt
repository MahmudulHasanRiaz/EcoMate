P47aj — Prisma relation alias fix (Product variants/comboItems + Staff payments + StaffIncome.staff)

Symptoms (from logs)
- /api/products fails: "Unknown field variants" on Product include
- /api/products fails: comboItems include expects child/variant
- /api/staff/clerk fails: include { payments: true }
- /api/staff aggregate fails: where { staff: ... }

Root cause
- prisma/schema.prisma relation field names still use legacy names (ProductVariant, ComboProductItem_..., StaffPayment, StaffMember) while code expects variants/comboItems/child/variant/payments/staff.

Fix (schema-only; no DB migration required)
Edit root prisma/schema.prisma (NOT src/prisma/schema.prisma):

A) Product model
Rename relation list fields to match code:
  ProductVariant ProductVariant[]
  -> variants ProductVariant[]

  ComboProductItem_ComboProductItem_parentIdToProduct ComboProductItem[]
  -> comboItems ComboProductItem[]

(Optional: keep the child-side list as comboChildren if needed)
  ComboProductItem_ComboProductItem_childIdToProduct ComboProductItem[]
  -> comboChildren ComboProductItem[]  (if referenced anywhere)

B) ComboProductItem model
Rename relation fields used by include { child, variant }:
  Product_ComboProductItem_childIdToProduct Product @relation(...)
  -> child Product @relation(...)

  ProductVariant ProductVariant? @relation(...)
  -> variant ProductVariant? @relation(...)

(Optional)
  Product_ComboProductItem_parentIdToProduct Product @relation(...)
  -> parent Product @relation(...)

C) StaffMember model
Rename StaffPayment list:
  StaffPayment StaffPayment[]
  -> payments StaffPayment[]

D) StaffIncome model
Rename StaffMember relation:
  StaffMember StaffMember @relation(...)
  -> staff StaffMember @relation(...)

2) Regenerate Prisma Client
- Local: npx prisma generate
- Docker: docker compose exec app npx prisma@6.19.1 generate

3) Restart dev/app

Expected result
- /api/products works (variants + comboItems includes)
- staff/clerk route works (payments include)
- staff aggregate works (where { staff: ... })
- Assigned-to-me filter can load staff list and resolve IDs
