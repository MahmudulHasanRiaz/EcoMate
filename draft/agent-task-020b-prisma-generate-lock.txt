UPDATE P20b - Fix Prisma generate EPERM (Windows file lock)

Scope: Local ops only. No code changes.

Symptom:
- npm run dev fails at `npx prisma generate` with EPERM rename of query_engine-windows.dll.node.

Cause:
- File lock from another Node/Prisma process (dev server, VS Code extension, or stuck node.exe).

Steps (safe order):
1) Stop dev server (Ctrl+C) and close any running terminals using the repo.
2) Close VS Code (or disable Prisma extension temporarily).
3) In PowerShell (run from repo root):
   Get-Process node | Stop-Process -Force
4) Run:
   npx prisma generate
5) Start dev server:
   npm run dev

Fallback (only if step 4 still fails):
- Delete the generated client folder and re-generate:
  Remove-Item -Recurse -Force node_modules\.prisma\client
  npx prisma generate

Deliver back:
- Confirmation Prisma generate succeeded
- Dev server starts normally
