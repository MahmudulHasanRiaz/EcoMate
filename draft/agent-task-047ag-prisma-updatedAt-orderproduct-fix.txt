P47ag — Prisma updatedAt + OrderProduct relation alias fix

Symptoms (from logs)
- staffMember.upsert fails: "Argument updatedAt is missing"
- order.findMany fails: "Unknown field product for select on OrderProduct"

Root cause
- prisma/schema.prisma (root) defines StaffMember.updatedAt without @updatedAt.
- OrderProduct relation fields are named Order/Product (capitalized), but code selects product.

Fix (schema-only, no DB changes)

1) Edit prisma/schema.prisma (root, NOT src/prisma/schema.prisma)

A) StaffMember.updatedAt
Change:
  updatedAt DateTime
To:
  updatedAt DateTime @updatedAt

B) OrderProduct relation field aliases
In model OrderProduct, rename fields:
  Order   Order   @relation(fields: [orderId], references: [id], onDelete: Cascade)
  Product Product @relation(fields: [productId], references: [id], onDelete: Cascade)
To:
  order   Order   @relation(fields: [orderId], references: [id], onDelete: Cascade)
  product Product @relation(fields: [productId], references: [id], onDelete: Cascade)

2) Regenerate Prisma client
- Local: npx prisma generate
- Docker: docker compose exec app npx prisma@6.19.1 generate

3) Restart dev/app process

Expected result
- Staff upsert no longer complains about updatedAt
- Orders API select { products: { product: ... } } works

Note
- These are Prisma field aliases only; no DB migration required.
