P47ay - Product delete crash (Product.attributes relation mismatch)

Problem
- Deleting a product throws: Unknown field `attributes` for include on model Product.
- Prisma relation is named Attribute (PascalCase), so `attributes` is invalid.

Scope
- src/services/products.ts

Fix
- In deleteProduct(), remove the invalid include or use the correct relation name.
- Since deleteProduct only needs image/id, safest is to select minimal fields.

Patch (exact edit)
Find:
  const product = await prisma.product.findUnique({
    where: { id: productId },
    include: { attributes: true },
  });

Replace with:
  const product = await prisma.product.findUnique({
    where: { id: productId },
    select: { id: true, image: true },
  });

Notes
- Do NOT use include: { attributes: true } (invalid).
- If you prefer to keep relation, use include: { Attribute: true } but it is unused.

Verify
- Delete a test product from /dashboard/products.
- No PrismaClientValidationError; delete completes.
