UPDATE P13b - Expense approval flow hardening (server + UI + print)

Scope: ONLY Expense module + print view. Keep changes minimal. No refactors.

Problems found:
1) Finance can mark Paid before approval (server doesn’t enforce).
2) New expense defaults to isPaid=true while status is Submitted, causing check validation.
3) paidById/paidByName sent from API but NOT saved in updateExpense().
4) Rejected expense does not clear check fields.
5) Print signature area is empty (no names/dates).

Fixes:

1) Enforce approval before payment (server)
File: src/app/api/expenses/[id]/route.ts
- Before allowing isPaid=true:
  const effectiveStatus = body.approvalStatus ?? existing.approvalStatus;
  if (body.isPaid === true && effectiveStatus !== 'Approved') {
     return NextResponse.json({ error: 'Expense must be Approved before payment' }, { status: 400 });
  }

2) Default new expense to unpaid (UI)
File: src/app/dashboard/expenses/client-page.tsx
- In initialFormValues for new expense:
  isPaid: false
  paidAt: '' (empty)
  check: 0
  checkDate: ''
  checkNo: ''
- Keep approvalStatus: 'Submitted'

3) Persist paidBy fields (server module)
File: src/server/modules/expenses.ts
- In updateExpense():
  if (typeof input.paidById !== 'undefined') (data as any).paidById = input.paidById || null;
  if (typeof input.paidByName !== 'undefined') (data as any).paidByName = input.paidByName || null;
- If input.isPaid === false, also set paidById/paidByName = null.

4) Clear check fields on rejection
File: src/server/modules/expenses.ts
- Inside approvalStatus === 'Rejected' block:
  data.check = 0;
  data.checkNo = null;
  data.checkDate = null;
  data.checkStatus = null;

5) Disable payment fields unless Approved (UI)
File: src/app/dashboard/expenses/expense-form.tsx
- Set isPaymentDisabled = approvalStatus !== 'Approved';
- Apply disabled={isPaymentDisabled} to:
  paidAt, paidFromAccountId, check, checkNo, checkDate inputs.

6) Print signature metadata (names + dates)
File: src/app/print/expenses/client-page.tsx
- In signature area, render:
  Prepared By: submittedByName + submittedAt (if present)
  Manager Approval: approvedByName + approvedAt
  Finance Payment: paidByName + paidAt
- Keep layout simple; only show values when available.

Deliver back:
- Summary + files touched
- Manual verify:
  - Cannot mark paid before approval (server returns 400)
  - New expense starts as Unpaid + Submitted
  - PaidBy shows in print view when paid
