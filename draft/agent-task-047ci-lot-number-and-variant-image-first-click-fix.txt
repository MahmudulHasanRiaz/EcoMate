Required:

     - `canonicalLotNumber`

   - Example where clause pattern:
     - `OR: [{ lotNumber: canonicalLotNumber }, { lotNumber: po.id }]`
3. Write-path standardization:
   - all new inventory writes must use canonical `PO-<id>` format.

Acceptance:

2) Fix variant image first-click + persistence across accordion switch
--------------------------------------------------------------------

Files:
- `src/app/dashboard/products/new/page.tsx`
- `src/app/dashboard/products/[id]/edit/page.tsx`

Root cause:
- For variant file selection, form field value is saved as `__file__:<variationId>`, but UI `images` source only reads URL string.
- So selected file preview disappears (looks like not selected) when re-rendering or switching accordion item.

Required implementation:
1. In each variant image `render` block, compute preview source with pending file support:
   - `const pendingFile = variantImageFilesRef.current[variationId];`
   - `const currentUrl = ...` (existing logic for real URL)
   - `const images = pendingFile ? [pendingFile] : (currentUrl ? [{ id: currentUrl, url: currentUrl }] : []);`
2. Keep existing behavior:
   - file upload -> `setVariantImageFile(variationId, file)` + `field.onChange(__file__ marker)`
   - library selection -> clear pending file + set direct URL
   - clear action -> clear both pending file and field value
3. Ensure switching to another variant accordion and back does NOT lose previously selected unsaved file previews.
