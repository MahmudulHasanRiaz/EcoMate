P47am - Category combobox mouse click fix (searchable + clickable)

Problem
- Category combobox search works with Enter, but mouse click on results does not select.
- Likely Popover portal + CommandItem pointer handling issue.

Scope
- src/components/products/category-combobox.tsx

Fix
1) Render PopoverContent without portal (prevents pointer-events/overlay issues):
   - Change <PopoverContent ...> to <PopoverContent ... portalled={false}>

2) Make selection fire on mouse as well as keyboard:
   - Add onMouseDown to prevent blur before select.
   - Add onClick to ensure mouse selection triggers even if cmdk onSelect is skipped.

Patch (exact edits)

A) Update PopoverContent opening tag:
- <PopoverContent className="w-[200px] p-0 shadow-md" align="start">
+ <PopoverContent className="w-[200px] p-0 shadow-md" align="start" portalled={false}>

B) Update CommandItem:
Replace current CommandItem with:

<CommandItem
  key={opt.id}
  value={`${opt.name}-${opt.id}`}
  onSelect={() => {
    onChange(opt.id);
    setOpen(false);
  }}
  onMouseDown={(e) => e.preventDefault()}
  onClick={() => {
    onChange(opt.id);
    setOpen(false);
  }}
  className={cn(
    "cursor-pointer",
    opt.level === 1 && "pl-8"
  )}
>

Notes
- The onMouseDown prevents the input blur from canceling selection.
- The onClick guarantees mouse selection even if cmdk onSelect is skipped.
- Enter/keyboard still works via onSelect.

Test
- Open Products page, open category dropdown.
- Type to filter.
- Click an item with mouse ? dropdown closes, filter changes.
- Press Enter on highlighted item ? still works.
