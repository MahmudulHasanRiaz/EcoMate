--------------------------------------------------
1) Guard against invalid payload IDs (critical correctness)
--------------------------------------------------
File: src/app/api/webhooks/woo/[id]/route.ts

Current risk:
- `const externalOrderId = String(payload.id)` can become `"undefined"` / `"null"` / empty.
- That can generate invalid local IDs and noisy failures.

Required fix:
- Validate payload order ID before processing:
  - accept only non-empty scalar (`string|number`) id
  - reject invalid values with `apiError('Invalid payload: missing order id', 400)`
- Do this before building `internalOrderId`.

Acceptance:
- webhook with missing/invalid `payload.id` must not call `processWooWebhookPayload`.

--------------------------------------------------
2) Remove sensitive signature data from logs (security)
--------------------------------------------------
File: src/app/api/webhooks/woo/[id]/route.ts

Current risk:
- mismatch log prints expected HMAC and received signature.

Required fix:
- Do NOT log full signature/hash values.
- Log only minimal metadata (integrationId + reason).

Acceptance:
- no log line should expose full computed or incoming signature.

--------------------------------------------------
3) Keep auto-heal, but make failure handling explicit and non-blocking
--------------------------------------------------
File: src/app/api/webhooks/woo/[id]/route.ts

Required behavior (keep):
- On missing secret/signature or mismatch:
  - record webhook failure
  - trigger `ensureWooWebhook(integration, { forceRecreate: true, rotateSecret: true })`
  - return 401

Required hardening:
- Keep auto-heal call wrapped in `.catch(...)` (already present) so request path never crashes on heal error.
