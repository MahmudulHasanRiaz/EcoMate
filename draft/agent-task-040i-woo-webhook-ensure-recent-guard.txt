UPDATE P40i - Prevent double create by skipping ensure when recent ensure exists

Scope: ONLY src/server/modules/woo-sync.ts. Minimal change.

Problem:
- Auto-heal (forceRecreate) may run just before manual sync. It creates a new webhook.
- Manual sync then calls ensureWooWebhook again and can create a second webhook if it doesn’t respect the recent ensure.

Fix:
1) Track recent ensure in woo-sync
File: src/server/modules/woo-sync.ts
- Add near markWebhookEnsured:
```
const wooWebhookEnsureRecentMap = new Map<string, number>();
const WOO_WEBHOOK_ENSURE_RECENT_TTL_MS = 5 * 60 * 1000;
```
- Update markWebhookEnsured to also set memory:
```
wooWebhookEnsureRecentMap.set(integrationId, Date.now());
```
- Add helper:
```
async function hasRecentEnsure(integrationId: string) {
  const now = Date.now();
  const mem = wooWebhookEnsureRecentMap.get(integrationId);
  if (mem && now - mem < WOO_WEBHOOK_ENSURE_RECENT_TTL_MS) return true;
  try {
    const { getRedisClient } = await import('@/server/queues/redis');
    const redis = getRedisClient();
    if (redis) {
      const v = await redis.get(`woo:webhook:last_ensure:${integrationId}`);
      if (v) return true;
    }
  } catch (e) {
    console.error('[WOO_ENSURE_RECENT_READ_ERR]', e);
  }
  return false;
}
```

2) Skip ensure if recently ensured (unless forceRecreate)
- At the start of ensureWooWebhook (after null checks, before markEnsureInflight):
```
if (!options.forceRecreate && await hasRecentEnsure(integration.id)) {
  console.log('[WOO_WEBHOOK_ENSURE_SKIP_RECENT]', integration.id);
  return;
}
```
