UPDATE P35b - Fix Woo push circuit breaker success logic (avoid premature reset)

Scope: ONLY src/server/modules/stock-sync.ts. Minimal change. No refactors.

Problem:
- recordWooPushSuccess() is called unconditionally after attemptPushTargets.
- This clears failure counters even when a push actually failed.
- Circuit breaker will never open in practice.

Fix:
1) Track failures inside attemptPushTargets
File: src/server/modules/stock-sync.ts
- Add a local flag in attemptPushTargets:
  let hadFailure = false;
- Set hadFailure = true whenever:
  - res.ok is false (WOO_STATUS_PUSH_FAIL)
  - any catch block logs WOO_STATUS_PUSH_ERR
  (Optional: also mark true in WOO_MANAGE_STOCK_CHECK_ERR if you want.)

- Return this flag:
  return { retryableFailure, hadFailure };

2) Only clear circuit on full success
File: src/server/modules/stock-sync.ts
- In pushStockStatusToIntegration:
  - Let attempt = await attemptPushTargets(...)
  - If retryableFailure triggers second attempt, overwrite attempt with the second result.
  - Only call recordWooPushSuccess(integration.id) if attempt.hadFailure === false.
  - If hadFailure true, do NOT clear circuit.

Pseudo:
```
let attempt = await attemptPushTargets(...);
if (attempt.retryableFailure) {
  const refreshedTargets = await fetchWooTargetsBySku(..., { forceRefresh: true });
  if (refreshedTargets.length) {
    attempt = await attemptPushTargets(...);
  }
}
if (!attempt.hadFailure) {
  await recordWooPushSuccess(integration.id);
}
```
