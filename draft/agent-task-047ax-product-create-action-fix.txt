P47ax - Product create server-action mismatch fix

Problem
- Production shows: "Server Action was not found" when creating a product.
- Cause: createProduct is a server action imported into a client component; after deploy, stale client bundle causes action ID mismatch.

Fix (code)
1) Add a POST handler in src/app/api/products/route.ts that calls the existing createProduct() server logic and enforces permissions.
2) Create a client-only service wrapper that calls the API via fetch.
3) Update the product create page to use the client wrapper (no server action call from the browser).
4) Ensure the client wrapper unwraps the standard ApiResponse (data.success) via handleApiResponse so failures don't show as success.

Files changed
- src/app/api/products/route.ts
  + import createProduct
  + export async function POST(req)
- src/services/products-client.ts (new)
  + createProductClient(formData) => POST /api/products + handleApiResponse
- src/app/dashboard/products/new/page.tsx
  + use createProductClient instead of createProduct server action

Why this fixes it
- The UI no longer depends on Next Server Actions for product creation, so there is no action-id mismatch after deploy.
- Proper response unwrapping avoids false-success toasts.

Local-only steps (coder)
1) Update src/services/products-client.ts to use handleApiResponse for unwrapping.
2) Ensure import { handleApiResponse } from '@/lib/api-helper' exists in that file.
