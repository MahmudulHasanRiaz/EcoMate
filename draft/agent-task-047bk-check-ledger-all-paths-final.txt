1) Fix `updateCheckStatusCore` settlement branch using posting origin
--------------------------------------------------------------------
Current issue:
- `Passed` currently always posts `Dr AP / Cr Supplier Advance`.
- This is correct only when pending was staged in Supplier Advance.
- For flows that did NOT stage pending there (PO add/edit), this is wrong.

Required implementation (inside `updateCheckStatusCore` ledger transaction):

A) Gather context BEFORE deleting postingGroup entries
1. Resolve accounts:
   - `apId` from `ACCOUNT_LABELS.accountsPayable`
   - `supplierAdvanceId` from `ACCOUNT_LABELS.supplierAdvance`
   - `sourceAccountId` = `fullPayment.paidFromAccountId || cash account`
2. Query existing postingGroup lines:
   - `existingEntries = findMany({ where: { postingGroup }, select: { accountId, debit, credit } })`
3. Detect staging:
   - `hadPendingAdvanceStage = existingEntries.some(e => e.accountId === supplierAdvanceId && e.debit > 0)`

B) Delete old postingGroup rows (keep current idempotent behavior)

C) Rebuild rows by target status

Case 1: `status === 'Passed'` and `fullPayment.check > 0`
- If `hadPendingAdvanceStage`:
  - post `Dr Accounts Payable`
  - post `Cr Supplier Advance`
- Else:
  - post `Dr Accounts Payable`
  - post `Cr sourceAccountId` (bank/cash/direct account)

Case 2: `status === 'Pending'` and `fullPayment.check > 0`
- Re-stage only when pending-stage model applies:
  - If `hadPendingAdvanceStage` OR payment is partner-FIFO-origin (see Rule D below):
    - post `Dr Supplier Advance`
    - post `Cr sourceAccountId`
- Else keep no posting for pending.

Case 3: `status in ('Bounced','Cancelled')`
- Keep no postingGroup rows (fully reversed).

D) Partner-FIFO origin rule (for re-pending after pass)
- Add helper boolean:
  - `isPartnerFifoPayment = fullPayment.paymentFor === 'Purchase Balance (FIFO)' || (fullPayment.paymentFor || '').startsWith('Production Step (')`
- Use this for Case 2 when `hadPendingAdvanceStage` is not detectable due to prior status transitions.

Acceptance:
- Partner FIFO path: Pending = staged in Supplier Advance; Passed = clears from Supplier Advance to AP.
- PO add/edit path: Pending (no staging) remains no-ledger; Passed = Dr AP / Cr source account.
- No double-credit bank in any status transition.
