P47am-v2 - Category combobox mouse click fix (cmdk value/keywords + portal)

Problem
- Search works and Enter selects, but mouse clicks on list items do not select.
- Current component uses cmdk with custom onClick/onMouseDown and non-portalled Popover; click still fails.

Scope
- src/components/products/category-combobox.tsx

Fix
1) Use cmdk canonical value/keywords so mouse activation uses cmdk internals:
   - value={opt.id}
   - keywords={[opt.name]}
   - onSelect receives id and drives onChange
2) Remove custom onClick/onMouseDown handlers (they can interfere with cmdk)
3) Re-enable portal rendering (default) so popover is not constrained by parent stacking/overflow.
4) Prevent focus loss before selection with onPointerDown on item.

Patch (exact edits)

A) PopoverContent: remove portalled={false}
- <PopoverContent className="w-[200px] p-0 shadow-md" align="start" portalled={false}>
+ <PopoverContent className="w-[200px] p-0 shadow-md" align="start">

B) CommandItem: replace props
Current:
<CommandItem
  key={opt.id}
  value={`${opt.name}-${opt.id}`}
  onSelect={() => { ... }}
  onMouseDown={(e) => e.preventDefault()}
  onClick={() => { ... }}
  className={...}
>

Replace with:
<CommandItem
  key={opt.id}
  value={opt.id}
  keywords={[opt.name]}
  onSelect={(currentValue) => {
    onChange(currentValue);
    setOpen(false);
  }}
  onPointerDown={(e) => e.preventDefault()}
  className={cn(
    "cursor-pointer",
    opt.level === 1 && "pl-8"
  )}
>

Notes
- keywords keeps search by name while value stays stable.
- onPointerDown prevents blur without overriding cmdk click.
- Portal avoids layout overlays swallowing pointer events.
