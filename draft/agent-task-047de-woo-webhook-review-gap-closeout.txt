Why this follow-up exists
- P47dd implementation is partially correct, but review found critical gaps that can still break auto-ingest stability.
- Manual sync path is now working, but webhook path must be made fully consistent with sync path and self-healing behavior.

Review findings to fix (ordered by severity)

1) CRITICAL - Order ID format mismatch in webhook route
File:
- src/app/api/webhooks/woo/[id]/route.ts

Current issue:
- Route calls processor with:
  - `internalOrderId = integrationId + ':' + externalOrderId`
- Sync module uses:
  - `woo-${integration.id}-${wo.id}`

Impact:
- Same Woo order can be stored under different IDs between webhook vs manual sync.
- Duplicate/skipped logic becomes unreliable.

Required fix:
- Build internal order ID exactly:
  - `const internalOrderId = \`woo-${integration.id}-${externalOrderId}\`;`
- Pass this to `processWooWebhookPayload(...)`.

2) HIGH - Missing webhook auto-heal on auth/config failures
File:
- src/app/api/webhooks/woo/[id]/route.ts

Current issue:
- On missing secret/signature mismatch, route returns 401 but does not auto-heal webhook registration.

Impact:
- If secret drifts or webhook config breaks, auto-ingest can silently stop for long periods.

Required fix:
- Import and use `ensureWooWebhook` from `@/server/modules/woo-sync`.
- On these cases:
  - missing webhookSecret
  - missing `x-wc-webhook-signature`
  - signature mismatch
- Must:
  - record failure
  - trigger `ensureWooWebhook(integration, { forceRecreate: true, rotateSecret: true })`
  - return 401

Note:
- Protect from storm via existing ensure lock/throttle already inside woo-sync.

3) MEDIUM - Insecure fallback: accepts unsigned payload when secret missing
File:
- src/app/api/webhooks/woo/[id]/route.ts

Current issue:
- If `integration.webhookSecret` absent, route logs warning and continues without signature verification.

Impact:
- Security hole and possible fake order ingestion.

Required fix:
- Do NOT accept unsigned payload when webhook secret is missing.
- Treat as configuration error:
  - record failure
  - auto-heal trigger
  - return 401
