UPDATE P40c - Woo webhook auto-heal hardening (throttle fallback + record failures)

Scope: ONLY src/app/api/webhooks/woo/[id]/route.ts. Minimal changes.

Problems to fix:
- Auto-heal throttle only works when Redis is available; without Redis it can run on every 401.
- 401 signature/missing-secret events are NOT recorded in Webhook Failures, so the new Failures UI is blind to these incidents.

Fixes:
1) Add in-memory throttle fallback
File: src/app/api/webhooks/woo/[id]/route.ts
- Above triggerWooWebhookAutoHeal, add module-level map:
  const wooWebhookAutoHealMap = new Map<string, number>();

- Update triggerWooWebhookAutoHeal to use in-memory throttle if Redis is unavailable:
  const now = Date.now();
  const last = wooWebhookAutoHealMap.get(integrationId);
  if (!redis && last && now - last < 30 * 60 * 1000) {
     console.log('[WOO_WEBHOOK_AUTOHEAL] Throttled (memory).', integrationId);
     return;
  }
  if (!redis) wooWebhookAutoHealMap.set(integrationId, now);

- Keep existing Redis throttle as-is.

2) Record 401 failures in Webhook Failures log
File: src/app/api/webhooks/woo/[id]/route.ts
- In the three 401 paths (missing webhookSecret, missing signature, invalid signature), before returning apiError:
  await recordWebhookFailure({
     source: 'woo-webhook',
     integrationId,
     orderId: internalOrderId,
     externalOrderId,
     payload: payloadText || undefined,
     error: new Error('Missing webhook secret' / 'Missing signature' / 'Invalid signature'),
  });
- Use a short error message string per case.
- Do NOT log secrets.

3) Keep behavior unchanged otherwise
- Still trigger auto-heal in those 401 cases.
- No changes to success path or processWooWebhookPayload.
