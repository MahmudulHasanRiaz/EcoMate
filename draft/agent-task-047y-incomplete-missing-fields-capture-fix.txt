P47y - Incomplete Capture: Missing Name/Address (phone-only payload fix)

Problem observed
- Customer fills name/address but incomplete lead contains only phone.
- Raw payload shows empty fields except phone/referrer/landingPage.

Root cause (very likely)
- `getVal()` only reads `:visible` elements.
- We trigger capture on `pagehide` / `visibilitychange` where fields are no longer considered visible.
- Result: phone is passed explicitly, but other fields are read as empty.

Scope
- Only update: draft/fashionary-woo-plugin/assets/js/checkout.js

----------------------------------------------------------------------
1) Make getVal robust to hidden fields (critical fix)
----------------------------------------------------------------------
Find:
```js
function getVal(selectors) {
    for (const sel of selectors) {
        const $el = $(sel).filter(':visible').first();
        if ($el.length) {
            const v = String($el.val() || '').trim();
            if (v) return v;
        }
    }
    return '';
}
```

Replace with:
```js
function getVal(selectors) {
    for (const sel of selectors) {
        // 1) Try visible element first
        let $el = $(sel).filter(':visible').first();

        // 2) Fallback to the first matching element even if hidden
        if (!$el.length) {
            $el = $(sel).first();
        }

        if ($el.length) {
            const v = String($el.val() || '').trim();
            if (v) return v;
        }
    }
    return '';
}
```
