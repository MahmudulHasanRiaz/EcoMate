P12 - Marketing + Finance Manager roles + Marketing reports + Expense approval flow

Scope: Planning + role implementation tasks. Keep changes modular.
Goal: Add Marketing + Finance Manager roles, create a Marketing Reports menu, and add an expense approval/print workflow. Keep ledger logic clean (no double-posting).

DECISION (Architect):
1) Staff Payment MUST create an Expense entry (Salary & Allowances) so it appears in expenses + print + approvals.
2) Ledger posting must be SINGLE-SOURCE:
   - Use Expense module to post ledger (Dr Expense, Cr AP; on payment Dr AP, Cr Cash/Bank).
   - StaffPayment route should NOT post ledger if it creates an Expense.
   - Keep staffPayment linked to expense via Expense.staffPayment.
3) Check Passing should rely on Expense check fields (and Purchase/Staff only if no expense exists).

Roles & Permissions (baseline):
1) Marketing Manager (label: "Marketer")
   - marketing: create/read/update (new module)
   - analytics: readOnly
   - orders: readOnly (for outcome review)
   - expenses: readOnly (only to view approved spend)
   - customers: readOnly (optional)
   - rest: noAccess
   - Page access: dashboard + marketing + analytics + orders (optional customers)

2) Finance Manager (label: "Finance Manager")
   - accounting: full (create/read/update)
   - expenses: full (create/read/update/pay)
   - checkPassing: full
   - staff: readOnly (to view staff payments)
   - purchases: readOnly (to verify WIP/Inventory cost)
   - analytics: readOnly (financial overview)
   - rest: noAccess
   - Page access: dashboard + expenses + accounting + checkPassing + staff + purchases + analytics

PHASE B - Marketing Reports menu (design + plan)
New menu: "Marketing" under dashboard.
Data model proposal:
- MarketingCampaign: id, name, businessId, channel, objective, startDate, endDate, budget, status
- MarketingSpend: id, campaignId, date, amount, notes, source (manual/import), expenseId (optional)
- MarketingOutcome: id, campaignId, date, metricType (orders/leads/revenue/custom), value

Key metrics:
- Total Spend, Total Results, CPR (Cost Per Result), CPA, ROAS (if revenue tracked)
- Filters: business, date range, channel, status

Data sources:
Primary (best for now): Explicit order attribution to Campaign (manual assignment in Order UI).
Secondary (optional): Auto-attribution via UTM/Coupon when available.

Attribution rule (DECIDED):
- Orders are the default "Result" metric.
- CPR = Total Spend / Orders (attributed).
- ROAS = Revenue / Spend (attributed).
- Profit = Revenue - COGS - Spend (use existing order COGS).

UI Pages:
1) Marketing Overview (cards + chart + table)
2) Campaign List (create/edit/close)
3) Campaign Details (spend + outcomes + CPR timeline)

API:
- GET /api/marketing/campaigns (cursor)
- POST /api/marketing/campaigns
- GET /api/marketing/campaigns/[id]
- POST /api/marketing/spend
- POST /api/marketing/outcome

PHASE C - Expense approval + print workflow (needed for signatures)
Changes:
- Expense status: Draft -> Submitted -> Approved -> Paid (Rejected optional)
- ApprovedBy, ApprovedAt, PaidBy, PaidAt
- Print voucher includes signature boxes:
  - Manager signature (ApprovedBy)
  - Finance signature (PaidBy)
- Only Finance Manager can mark Paid.
- Manager can submit/approve.
Ledger:
1) On Submitted/Approved: create AP entry (Dr Expense, Cr AP)
2) On Paid: Dr AP, Cr Cash/Bank
3) If Rejected/Deleted: reverse AP entry
Check passing:
- Only when Paid via check -> create CheckPassingItem (Pending)

PHASE D - Staff Payment -> Expense integration (implement)
Flow:
- When staff payment is created:
  - Create Expense (category: Salary & Allowances, amount, staffPaymentId)
  - If check info present, store in expense as well
  - Use Expense module to post ledger
- Remove ledger posting from staff payment route OR skip ledger if expense created
- Update analytics: rely on Expense only (already noted in analytics.ts comments)

DELIVERABLES:

Spend approval rule (DECIDED):
- Marketer can create spend entries, but they are created as Expense = Draft/Submitted.
- Manager/Finance approve -> Paid (after signature + verification).
- Only Approved/Paid spend counts toward KPIs.
