B) Variable product variant-image stability fix (critical UX bug)
--------------------------------------------------------------------

Root cause summary
- In both new/edit pages, variant image FormField uses `__file__:<variationId>` marker.
- UI preview source is derived only from URL string; file selections are not persisted in render state.
- `ImageUploader` syncs internal state from props; props become empty for marker values, so first upload preview disappears.
- Accordion open/close remount causes the same issue across variants.

Required implementation

1) Add stable per-variation draft image state (both create & edit pages)
- Add state:
  - `variantImageDrafts: Record<string, File | { id: string; url: string }>`
- Keep existing `variantImageFilesRef` for submit payload.

2) Render priority for each variation image uploader
For each variation row:
- `variationId = form.getValues('variations.${index}.id')`
- `draft = variantImageDrafts[variationId]`
- `currentUrl = field.value` when it is a real URL (not marker)
- Pass `images` to `ImageUploader` using priority:
  1. draft (file or object) if exists
  2. else URL object from currentUrl
  3. else empty array

3) Update onImagesChange behavior
When user changes variant image:
- If cleared:
  - remove from `variantImageFilesRef`
  - remove from `variantImageDrafts`
  - `field.onChange('')`
- If selected File:
  - store in `variantImageFilesRef[variationId]`
  - store same `File` in `variantImageDrafts[variationId]`
  - `field.onChange('__file__:<variationId>')` (keep backend compatibility)
- If selected media-library URL object:
  - remove from `variantImageFilesRef`
  - store URL object in `variantImageDrafts`
  - `field.onChange(url)`

4) Prune stale draft map entries on variation regenerate/remove
- On `variationFields` change, remove drafts/files for IDs no longer present.
- Prevent orphan draft data.

5) Apply identical logic in both pages
- `src/app/dashboard/products/new/page.tsx`
- `src/app/dashboard/products/[id]/edit/page.tsx`

6) Do not break server contract
- Keep submit behavior:
  - append `variantImage:<variationId>` file entries in FormData
  - keep URL-based images unchanged
- No backend contract change required for this fix.
