P47al — Prisma WooCheckoutLead alias fix + AppSetting updatedAt default

Symptoms (from logs)
- /api/orders/incomplete 500: "Unknown field business" on WooCheckoutLead include.
- stock sync upsert fails: "Argument updatedAt is missing" for AppSetting.

Root cause
- WooCheckoutLead relations are still named Business/WooCommerceIntegration (capitalized), but code uses business/integration.
- AppSetting.updatedAt has no @updatedAt, so create requires explicit updatedAt.

Fix (schema-only; no DB migration required)
Edit root prisma/schema.prisma:

A) WooCheckoutLead model
Rename relation fields:
  Business Business @relation(...)
  -> business Business @relation(...)

  WooCommerceIntegration WooCommerceIntegration @relation(...)
  -> integration WooCommerceIntegration @relation(...)

B) AppSetting model
Set updatedAt to auto-update:
  updatedAt DateTime
  -> updatedAt DateTime @updatedAt

2) Regenerate Prisma client
- Local: npx prisma generate
- Docker: docker compose exec app npx prisma@6.19.1 generate

3) Restart dev/app

Expected result
- /api/orders/incomplete works (include { business, integration })
- stock audit setting upsert no longer errors
