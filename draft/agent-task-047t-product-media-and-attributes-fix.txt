P47t - Product Media Library 404 + Attribute Persistence Fix

Goal
- Media library থেকে select করলে 404 না হয়.
- Product edit করলে attributes wipe হয়ে না যায়, এবং attributes persist করে.

Root cause summary (tell coder)
1) getProductById() attributes include করছে না, ফলে edit page attributes = [] হয়ে server-এ যায়, আর server rebuild logic আগের attributes delete করে দেয়.
2) /api/products/images raw image URL return করছে (absolute/old domain/relative without slash), যেগুলো normalize না হয়ে media library-তে 404 তৈরি করে.

2) Service: getProductById must include attributes
File: src/services/products.ts

In getProductById() -> prisma.product.findUnique include block, add attributes: true

From:

```ts
include: {
  variants: true,
```

To:

```ts
include: {
  variants: true,

  comboItems: { include: { child: true, variant: true } },

```

Then in the return object of getProductById, add attributes mapping:

```ts
attributes:
  (product as any).attributes?.map((attr: any) => ({
    id: attr.id,
    name: attr.name,
    options: Array.isArray(attr.options) ? attr.options : [],
  })) || [],
```

Place it near other top-level fields like images/categoryId.

3) Edit page: hydrate attributes into form.reset

Inside the `if (productData) { form.reset({ ... }) }` block, add attributes:

4) Media library: normalize URLs in /api/products/images
File: src/app/api/products/images/route.ts

Add a local normalize helper near the top (after DEFAULT imports):

```ts
const DEFAULT_PLACEHOLDER = '/placeholder.svg';

function normalizeImageUrl(input?: string | null) {
  if (!input) return DEFAULT_PLACEHOLDER;
  const trimmed = String(input).trim();
  if (!trimmed) return DEFAULT_PLACEHOLDER;

  // Absolute URL: try to extract a safe pathname
  if (/^https?:\/\//i.test(trimmed)) {
    try {
      const u = new URL(trimmed);
      // If it contains /uploads, prefer the pathname
      if (u.pathname?.startsWith('/uploads/')) {
        return u.pathname;
      }
      return trimmed;
    } catch {
      return trimmed;
    }
  }

  // Local path but missing leading slash
  return trimmed.startsWith('/') ? trimmed : `/${trimmed.replace(/^\/+/, '')}`;
}
```

Now update URL collection logic:

Replace:

```ts
images.forEach(img => imageUrls.add(img.url));
```

With:

```ts
images.forEach((img) => {
  const normalized = normalizeImageUrl(img?.url);
  if (normalized && normalized !== DEFAULT_PLACEHOLDER) {
    imageUrls.add(normalized);
  }
});
```

And in the catch block replace:

```ts
if (p.image.trim()) {
  imageUrls.add(p.image);
}
```

With:

```ts
const normalized = normalizeImageUrl(p.image);
if (normalized && normalized !== DEFAULT_PLACEHOLDER) {
  imageUrls.add(normalized);
}
```
