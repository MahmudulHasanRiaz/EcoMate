P47bt - Fix production-only "Server Action ... not found" on Purchase Create

Problem
- Live shows:
  "Creation Failed - Server Action '<id>' was not found on the server"
  while local works.

Root cause
- `src/app/dashboard/purchases/new/page.tsx` is a client component calling server action `createPurchaseOrder` from `../actions`.
- Server Action IDs are build-specific. After deploy, users can still have old JS bundle (old action id) in browser cache, so server cannot resolve it.

Goal
- Remove direct Server Action call from Purchase Create UI.
- Use stable API route (`POST /api/purchases`) + client fetch wrapper.

Scope
- `src/app/api/purchases/route.ts`
- `src/services/purchases-client.ts` (new)
- `src/app/dashboard/purchases/new/page.tsx`

--------------------------------------------------------------------
1) Add POST API for purchase create
--------------------------------------------------------------------

File: `src/app/api/purchases/route.ts`

A) Add imports:
- `createPurchaseOrderCore` from `@/server/modules/purchases`
- `apiError` from `@/lib/error`

B) Add `POST(req: NextRequest)` handler:
- permission: `enforcePermission('purchases', 'create')`
- parse JSON body as payload
- call `createPurchaseOrderCore(payload)`
- if result.success is false -> return `apiError(result.message || 'Creation failed', 400)`
- if success -> return `apiSuccess(result, 'Purchase order created', 201)`
- catch -> `apiServerError(error)`

Note:
- Keep existing `GET` unchanged.

--------------------------------------------------------------------
2) Add client wrapper for purchase create
--------------------------------------------------------------------

Create file: `src/services/purchases-client.ts`

Implement:
```ts
import { handleApiResponse } from '@/lib/api-helper';

type CreatePurchaseOrderResult = {
  success: boolean;
  message?: string;
  poId?: string;
};

export async function createPurchaseOrderClient(payload: unknown): Promise<CreatePurchaseOrderResult> {
  try {
    const response = await fetch('/api/purchases', {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify(payload),
    });

    return await handleApiResponse<CreatePurchaseOrderResult>(response);
  } catch (error: any) {
    return { success: false, message: error?.message || 'Network error occurred.' };
  }
}
```

--------------------------------------------------------------------
3) Switch purchase new page to API client wrapper
--------------------------------------------------------------------

File: `src/app/dashboard/purchases/new/page.tsx`

B) In `handleCreatePurchase`, replace:
