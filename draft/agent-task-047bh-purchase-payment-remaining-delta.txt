1) Pending check must NOT mark PO/Step as paid
--------------------------------------------------------------------
Problem:
- In `applyPartnerPaymentCore`, status update currently uses:
  `newTotalPaid = paid + allocation`
- This incorrectly treats a newly created Pending check as cleared payment.

File:
- `src/server/modules/purchases.ts`

Function:
- `applyPartnerPaymentCore`

Required fix:
- Keep allocation writing logic as is (cash/check rows can still be created).
- Change status math so ONLY cash-like allocations affect `newTotalPaid`.

Implementation rule:
- `isCheck = method === 'Check'`
- `isCashLike = ['Cash', 'Direct', 'bKash', 'Nagad', 'Rocket'].includes(method)`
- `effectiveAllocation = isCashLike ? allocation : 0`
- `newTotalPaid = paid + effectiveAllocation`

Apply this in BOTH flows:
- Supplier PO update block
- Vendor step update block

Expected behavior:
- Check payment with status `Pending` creates payment/check-passing item
  but does not flip `paymentStatus` to Paid/Partial prematurely.
- Status changes after check is marked `Passed` through existing reconciliation flow.
