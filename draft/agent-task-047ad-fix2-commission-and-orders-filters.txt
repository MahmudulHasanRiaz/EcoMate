P47ad-FIX2 — Delivered-only commission consistency + Orders list search + Assigned-to-Me filter

Scope
- src/server/modules/orders.ts
- src/app/dashboard/orders/client-page.tsx

A) Commission consistency in updateOrderStatusDirect
Problem
- updateOrderStatusDirect still gives confirm commission immediately.
- It also does not delete commission on Canceled/Returned/Damaged.
This breaks the “commission only on Delivered” rule.

Fix
1) In updateOrderStatusDirect, REMOVE the immediate Confirmed commission block:
   - Remove the block that does recordStaffIncome() when shouldSetConfirmedBy && isExternalOrderSource(orderSource).
2) After the status update (same pattern as updateOrderStatus), add:
   - If normalizedStatus === 'Delivered', call awardCommissionOnDelivered(tx, order.id) inside a transaction.
   - If normalizedStatus is in ['Canceled','Returned','Damaged'], delete staffIncome for this order.

Pseudo:
if (normalizedStatus === 'Delivered') {
  await prisma.$transaction(async (tx) => {
    await awardCommissionOnDelivered(tx, order.id);
  });
}
if (normalizedStatus && ['Canceled','Returned','Damaged'].includes(normalizedStatus)) {
  await prisma.staffIncome.deleteMany({ where: { orderId: order.id } });
}

B) Orders list search bar not working
Problem
- debouncedSearch is never updated from searchTerm.

Fix (client)
Add debounce effect:

React.useEffect(() => {
  const term = searchTerm.trim();
  const t = window.setTimeout(() => {
    setDebouncedSearch(term);
    setNextCursor(null);
  }, 350);
  return () => window.clearTimeout(t);
}, [searchTerm]);

(Keep buildKey using debouncedSearch as it already does.)

C) “Assigned to Me” filter not catching current user
Problem
- Assigned-to-me uses currentStaffId from loggedInStaff/publicMetadata. If missing, assignedToId isn’t sent.

Fix (client)
1) Compute a single effective staff id:

const staffIdForFilters = currentStaffData?.id || loggedInStaff?.id || (user?.publicMetadata?.staffId as string | undefined);

2) Replace currentStaffId usage in buildKey / getOrders / preload / handleLoadMore with staffIdForFilters.

3) If assigneeFilter === 'me' and staffIdForFilters is empty, show a toast and reset to 'all':

React.useEffect(() => {
  if (assigneeFilter === 'me' && !staffIdForFilters) {
    toast({ variant: 'destructive', title: 'Staff ID missing', description: 'Assigned-to-Me filter needs your staff profile.' });
    setAssigneeFilter('all');
  }
}, [assigneeFilter, staffIdForFilters]);

Notes
- Do NOT backfill old commissions; only apply on status change going forward.
- Keep Delivered-only rule consistent across updateOrderStatus and updateOrderStatusDirect.
