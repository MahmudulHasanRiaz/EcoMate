P47cd - Marketing module hardening (conversion-focused + platform-wise + strict role access)

Objective
- Keep Marketing module simple and purchase-conversion centric.
- Track performance by platform: Meta, TikTok, Google.
- Enforce strict access scope:
  - Allowed roles only: `Admin`, `Manager`, `Marketer`, `Finance Manager`, `Project Manager`
  - Other roles must be denied from all marketing routes/pages.
- No unrelated metrics or over-engineering.

Scope
- `prisma/schema.prisma`
- `prisma/migrations/<timestamp>_marketing_conversion_platforms/migration.sql`
- `src/server/modules/marketing.ts`
- `src/app/api/marketing/overview/route.ts`
- `src/app/api/marketing/campaigns/route.ts`
- `src/app/api/marketing/campaigns/[id]/route.ts`
- `src/app/api/marketing/campaigns/[id]/spend/route.ts`
- `src/app/dashboard/marketing/page.tsx`
- `src/app/dashboard/marketing/campaign-details.tsx`
- `src/types/index.ts`
- permission mapping files:
  - `src/lib/permissions.ts`
  - `src/lib/page-access.ts`

--------------------------------------------------------------------
1) Data model updates (minimal, conversion-centric)
--------------------------------------------------------------------

Add enum:
- `MarketingPlatform` = `Meta | TikTok | Google`

Update `MarketingCampaign`:
- `platform MarketingPlatform`
- `targetCpaBdt Float?`
- `targetCpaUsd Float?`
- `endedAt DateTime?` (when campaign is switched off)

Update `MarketingSpend`:
- `amountBdt Float` (required)
- `amountUsd Float?` (optional)
- `fxRate Float?` (optional snapshot)
- Keep old `amount` for compatibility temporarily, but server must write/read from `amountBdt`.

Migration requirements:
- Backfill existing campaign platform to a safe default (`Meta`) only where null.
- Backfill `amountBdt = amount` for old spend rows.
- Add non-null constraints only after backfill.

--------------------------------------------------------------------
2) Access control hard rule (must)
--------------------------------------------------------------------

Enforce allowed roles at API/page level:
- `Admin`, `Manager`, `Project Manager`, `Marketer`, `Finance Manager`

Implementation notes:
- Keep permission checks (`marketing.read/update/...`) as primary.
- Add role guard fallback in marketing API routes and marketing page loader.
- If role not in allowed list, return 403 with clear JSON message.

--------------------------------------------------------------------
3) Admin-only sensitive profitability
--------------------------------------------------------------------

Business rule:
- Actual cost-based profit (`Revenue - COGS - Spend`) is visible only to Admin.
- Non-admin (including Marketer/Manager/Project Manager/Finance Manager):
  - can see Spend, Purchases, CPA, Revenue, Target CPA, Gap vs Target
  - cannot see actual profit amount from COGS.

Server-side enforcement (mandatory):
- In `getMarketingOverview`, `getCampaigns`, `getCampaignDetails`, accept actor context.
- If actor is not Admin:
  - do not return `profit`, `totalProfit`, or COGS-based fields.
  - return `null` or omit these fields consistently.

UI:
- Show profit cards/columns only for Admin.
- For others, show Target vs Actual CPA cards/tables.

--------------------------------------------------------------------
4) Conversion-focused KPI definitions
--------------------------------------------------------------------

Use only these core KPIs:
- Spend (BDT primary, USD optional display)
- Attributed Purchases (valid order status only)
- CPA = Spend / Purchases
- Revenue (attributed)
- Target CPA (campaign)
- CPA Gap = Actual CPA - Target CPA
- Target Hit % (optional; if target set)

Do not add vanity metrics (engagement/click/impression/etc.).

--------------------------------------------------------------------
5) Platform-wise reporting
--------------------------------------------------------------------

Add platform performance aggregation (overview endpoint):
- Group by platform (Meta, TikTok, Google):
  - total spend
  - total purchases
  - CPA
  - revenue
  - (Admin only) actual profit

UI requirement:
- Add a clean table/card section: "Platform Performance"
- Must work with existing date/business filters.

--------------------------------------------------------------------
6) Campaign lifecycle + budget visibility
--------------------------------------------------------------------

Campaign create/edit:
- marketer sets:
  - platform
  - startDate
  - optional endedAt
  - budget
  - targetCpaBdt (required for campaign quality)
  - targetCpaUsd (optional)

Campaign listing/details show:
- budget
- total spent
- remaining budget = budget - spent
- campaign on/off status inferred from status + endedAt

If budget changes later:
- update campaign record directly; no separate audit table in this phase.

10) Verification checklist (mandatory)
--------------------------------------------------------------------

Role access:
1. Admin can open marketing page and all endpoints.
2. Marketer/Manager/Project Manager/Finance Manager can access.
3. Any other role gets 403.

Profit visibility:
1. As Admin: profit values visible in API + UI.
2. As Marketer: profit values hidden/null in API and not rendered in UI.

Platform metrics:
1. Create campaigns for Meta/TikTok/Google.
2. Add spends and attributions.
3. Platform table totals match campaign totals.

Currency:
1. Add spend with BDT only -> save succeeds.
2. Add spend with BDT+USD -> both persist.
3. CPA uses BDT as primary.

Target tracking:
1. Set target CPA on campaign.
2. Report shows actual CPA + gap correctly.

Date presets:
1. This Week / This Month / Last 30 Days produce expected filtered totals.

Responsive sanity:
1. Marketing page and campaign details usable on mobile + desktop without layout break.
