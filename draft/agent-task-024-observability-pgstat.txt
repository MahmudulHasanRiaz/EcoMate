UPDATE P24 - Observability baseline via pg_stat_statements (DB-only)

Scope: Database observability only. No app code changes.
Goal: Enable pg_stat_statements and capture a baseline slow-query report.

Steps (run on DB as superuser):
1) Check preload libs:
   SHOW shared_preload_libraries;
   - If pg_stat_statements not listed, add it:
     a) postgresql.conf -> shared_preload_libraries = 'pg_stat_statements'
     b) restart Postgres service
   (Alternative, if allowed: ALTER SYSTEM SET shared_preload_libraries = 'pg_stat_statements'; then restart.)

2) Enable extension:
   CREATE EXTENSION IF NOT EXISTS pg_stat_statements;

3) Capture top queries (write to file):
   Run SQL below and save output to draft/pg-stat-report.txt with timestamp header.

SQL:
   SELECT now() AS captured_at,
          calls,
          total_exec_time,
          mean_exec_time,
          rows,
          query
   FROM pg_stat_statements
   WHERE dbid = (SELECT oid FROM pg_database WHERE datname = current_database())
   ORDER BY total_exec_time DESC
   LIMIT 20;

4) If permission denied or extension unavailable:
   - Report exact error message in draft/pg-stat-report.txt

Deliver back:
- draft/pg-stat-report.txt (with output)
- Confirmation of Postgres restart (if required)
- Note any permission blockers
