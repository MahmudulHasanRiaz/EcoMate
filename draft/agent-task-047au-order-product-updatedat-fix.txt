P47au - Manual order create fails (OrderProduct.updatedAt missing)

Issue
- Prisma validation error during order.create when creating nested products.
- OrderProduct.updatedAt has no default and must be provided on create.

Scope
- src/server/modules/orders.ts

Fix
1) Create a single `now` timestamp and pass it to each OrderProduct create item.

Patch
Find the productCreates block in createOrder:

// Consolidate input items (handle both 'items' or 'products' from frontend)
const inputItems = Array.isArray(data.items) ? data.items : (Array.isArray(data.products) ? data.products : []);

const productCreates = inputItems
  .map((p: any) => ({
    productId: p.productId,
    variantId: p.variantId || null,
    quantity: Number(p.quantity || 0),
    price: Number(p.price || 0),
    siteDiscount: Number(p.siteDiscount || 0),
  }))
  .filter((p: any) => p.productId && p.quantity > 0);

Change to:

const inputItems = Array.isArray(data.items) ? data.items : (Array.isArray(data.products) ? data.products : []);
const now = new Date();

const productCreates = inputItems
  .map((p: any) => ({
    productId: p.productId,
    variantId: p.variantId || null,
    quantity: Number(p.quantity || 0),
    price: Number(p.price || 0),
    siteDiscount: Number(p.siteDiscount || 0),
    updatedAt: now,
  }))
  .filter((p: any) => p.productId && p.quantity > 0);

Notes
- Do NOT add updatedAt anywhere else; this fixes nested create only.

Restart
- npm run dev
- npm run worker

Verify
- Login as Seller/Moderator
- Create a manual order
- Should succeed without Prisma validation error
