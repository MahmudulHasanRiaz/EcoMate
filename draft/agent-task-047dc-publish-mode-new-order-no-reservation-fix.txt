P47dc - Publish Mode Guard: Prevent Stock Reservation on Woo New Orders

Problem
- In "No (Published Based)" mode, Woo orders imported with status New are still reserving stock before Confirmed.
- Expected behavior: In publish mode, New should NOT reserve stock. Stock movement should happen only on Confirmed/Damaged flow.

Root cause (code-level)
- Guard exists in `src/server/modules/woo-sync.ts`.
- Missing in `src/server/modules/woo/webhook-processor.ts` where New orders still call:
  - `handleStockReservation`
  - `handleComboStockReservation`

Scope (strict)
- src/server/modules/woo/webhook-processor.ts
- scripts/release-woo-new-reservations-publish.ts (new one-off repair script)

Do NOT touch unrelated files.

--------------------------------------------------
1) Fix webhook path reservation logic
--------------------------------------------------
File: src/server/modules/woo/webhook-processor.ts

Required changes:
1. Add stock mode helper in this file:
   - Read general settings via `getGeneralSettings()` from `@/server/utils/app-settings`.
   - return `'publish' | 'inventory'`.

2. In reservation block (currently around `if (saved.status === 'New' && !saved.isStockReserved)`) add mode guard:
   - Only reserve if `mode !== 'publish'`.
   - In publish mode, skip reservation and log:
     - `[STOCK_RESERVE_SKIP] Publish mode active, skipping reservation for Webhook order <id>`

3. Do not change status mapping (New/Draft logic remains unchanged).

Acceptance:
- In publish mode, webhook-created New orders must have `isStockReserved = false` and no reservation logs.

--------------------------------------------------
2) Add one-off repair script for already-reserved Woo New orders
--------------------------------------------------
File: scripts/release-woo-new-reservations-publish.ts (new)

Purpose:
- For already imported Woo New orders where `isStockReserved=true` in publish mode, release reservation and set flag false.

Script behavior:
1. Read general settings; if stockSyncMode != publish => exit without changes.
2. Find orders:
   - source='woo'
   - status='New'
   - isStockReserved=true
   - isStockDeducted=false
3. For each order in transaction:
   - load full products + product/variant/combo relations (same include shape used by reservation functions)
   - call:
     - `handleStockReservationRelease(tx, order, 'System-Fix')`
     - `handleComboStockReservationRelease(tx, order, 'System-Fix')`
   - update order `isStockReserved=false`
4. Print summary:
   - scanned
   - fixed
   - failed ids (if any)

Safety:
- idempotent: second run should fix 0.
