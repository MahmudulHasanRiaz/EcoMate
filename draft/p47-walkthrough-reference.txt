# Walkthrough - Staff Commission & Performance Enhancements

I have implemented the new commission logic where staff commissions are now awarded **only when an order reaches 'Delivered' status**. I also enhanced staff performance tracking with new pipeline ratios.

## Key Changes

### 1. Commission Model Update
Added `onOrderConvert` commission rate to handle incomplete lead conversions.
- Modified [staff-compensation.ts](file:///e:/fashionary/Fashionary/src/server/utils/staff-compensation.ts) to include the new field in normalization.
- Updated [staff.ts](file:///e:/fashionary/Fashionary/src/lib/validations/staff.ts) validations.
- Updated [index.ts](file:///e:/fashionary/Fashionary/src/types/index.ts) type definitions.

### 2. Backend Commission Logic ([orders.ts](file:///e:/fashionary/Fashionary/src/server/modules/orders.ts))
- **Deferred Payout**: Immediate commission awards on 'Created' and 'Confirmed' actions have been removed.
- **`awardCommissionOnDelivered`**: A new helper function ensures that 'Created', 'Confirmed', and 'Converted' commissions are only recorded when an order's status is updated to `Delivered`.
- **Automatic Cleanup**: If an order status changes to `Canceled`, `Returned`, or `Damaged`, all associated `StaffIncome` entries (commissions) are automatically deleted.

### 3. Lead Conversion Attribution ([route.ts](file:///e:/fashionary/Fashionary/src/app/api/orders/incomplete/%5Bid%5D/convert/route.ts))
- Incomplete lead conversions now correctly attribute `createdBy` and `assignedToId` to the staff member performing the conversion, ensuring they receive the `onOrderConvert` commission once the order is delivered.

### 4. Performance Pipeline Ratios
- Updated [getStaffPerformance](file:///e:/fashionary/Fashionary/src/server/utils/staff-performance.ts) to return separate order status breakdowns for orders **Created** vs **Confirmed** by staff.
- Calculated three new pipeline success ratios:
    - **Creation Pipeline**: Success rate of orders created by the staff.
    - **Confirmation Pipeline**: Success rate of orders confirmed by the staff.
    - **Total Pipeline**: Combined success rate across both pipelines.

### 5. Frontend UI Updates
- **Staff Form**: Added input for "On Order Convert (Incomplete)" commission rate in [dashboard/staff/page.tsx](file:///e:/fashionary/Fashionary/src/app/dashboard/staff/page.tsx).
- **Staff Profile & Account**: 
    - Displayed the new commission setting in the Payment Details summary.
    - Added new ratio cards in the Performance Overview section in both [staff/[id]/page.tsx](file:///e:/fashionary/Fashionary/src/app/dashboard/staff/[id]/page.tsx) and [account/page.tsx](file:///e:/fashionary/Fashionary/src/app/dashboard/account/page.tsx).
