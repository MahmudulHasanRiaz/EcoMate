UPDATE P40e - Woo webhook secret rotation on auto-heal (prevent hidden mismatch)

Scope: ONLY src/server/modules/woo-sync.ts and src/app/api/webhooks/woo/[id]/route.ts. Minimal change.

Goal:
- When auto-heal runs (forceRecreate), rotate webhook secret so all stale deliveries fail and only the new hook succeeds.
- Prevent silent mismatch if Woo keeps older secret in queue.

Changes:
1) Add option to rotate secret on forceRecreate
File: src/server/modules/woo-sync.ts
- Extend ensureWooWebhook options to accept rotateSecret?: boolean
  export async function ensureWooWebhook(integration: any, options: { forceRecreate?: boolean; rotateSecret?: boolean } = {}) {
- If options.rotateSecret === true:
  - Generate new secret (whsec_...) even if integration.webhookSecret already exists.
  - Update DB with new secret.
  - Invalidate Redis cache (same as existing secret generation path).
  - Update local integration.webhookSecret.
  - Log: [WOO_WEBHOOK_SECRET_ROTATED] integrationId
- Keep existing behavior if rotateSecret is not set.

2) Use rotateSecret when auto-heal triggers
File: src/app/api/webhooks/woo/[id]/route.ts
- In triggerWooWebhookAutoHeal, change:
  await ensureWooWebhook(integration, { forceRecreate: true });
  to:
  await ensureWooWebhook(integration, { forceRecreate: true, rotateSecret: true });

3) Safeguards
- Do NOT rotate secret on normal (non-autoheal) ensure call.
- Only rotate when autoheal triggers from 401 conditions.
