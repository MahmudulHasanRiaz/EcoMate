UPDATE P15 - Fix infinite render loop in purchase details (inventoryLots sync)

Scope: ONLY src/app/dashboard/purchases/[id]/client-page.tsx. Keep changes minimal. No refactors.

Problem:
Maximum update depth exceeded at useEffect syncing inventoryLots.
Cause: inventoryLots has a default [] in destructuring, creating a new array every render -> useEffect runs -> setState -> render loop.

Fix:
1) Remove default `inventoryLots = []` from component props destructuring.
   - Leave the prop optional in the type.

2) Initialize local state with a stable fallback once:
   Change:
     const [localInventoryLots, setLocalInventoryLots] = useState<InventoryItem[]>(inventoryLots || []);
   To:
     const [localInventoryLots, setLocalInventoryLots] = useState<InventoryItem[]>(() => inventoryLots ?? []);

3) Guard the sync effect to only run when inventoryLots is provided:
   Change:
     useEffect(() => {
       if (inventoryLots) setLocalInventoryLots(inventoryLots);
     }, [inventoryLots]);
   To:
     useEffect(() => {
       if (!Array.isArray(inventoryLots)) return;
       setLocalInventoryLots(inventoryLots);
     }, [inventoryLots]);

Verify:
1) Load purchase details page; no "Maximum update depth exceeded".
2) inventoryLots still appears when provided, and refreshInventoryLots works.
