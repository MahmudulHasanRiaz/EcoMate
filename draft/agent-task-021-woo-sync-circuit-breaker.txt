Goal:

1) Add helpers (woo-sync.ts)
Add near top (after imports):

Constants:
- WOO_RATE_GAP_MS = 500
- WOO_CB_FAIL_WINDOW_MS = 5 * 60 * 1000
- WOO_CB_FAIL_THRESHOLD = 3
- WOO_CB_OPEN_MS = 10 * 60 * 1000
- WOO_LOCK_TTL_MS = 2 * 60 * 1000

State + helpers:
- wooRateMap: Map<string, number>
- buildWooKey(...parts): `woo:sync:${parts.join(':')}`

Functions:
- async wooRateLimit(integrationId)
  - Use Redis (if available) with key buildWooKey('rate', integrationId)
  - If last call < WOO_RATE_GAP_MS, wait
  - Set last timestamp with PX
  - Fallback to in-memory map if no Redis

- async isCircuitOpen(integrationId): boolean
  - Redis key: buildWooKey('cb:open', integrationId)
  - Return true if exists

- async recordWooFailure(integrationId)
  - Redis INCR key buildWooKey('cb:fail', integrationId)
  - Set PX to WOO_CB_FAIL_WINDOW_MS on first increment
  - If count >= WOO_CB_FAIL_THRESHOLD => set buildWooKey('cb:open', integrationId) with PX = WOO_CB_OPEN_MS
  - Fallback: use in-memory counter map with timestamps if Redis missing

- async clearWooFailures(integrationId)
  - Redis DEL fail + open keys (or reset in-memory)

- async withWooIntegrationLock(integrationId, task)
  - Redis lock key buildWooKey('lock', integrationId)
  - Use token + NX + PX WOO_LOCK_TTL_MS, retry every 200ms until acquired
  - Release with compare-and-del (Lua)
  - If no Redis, just run task()
